## 📌 Project Overview
This project involves a comprehensive analysis of the "SuperStore" dataset. The goal was to transform a flat data structure into a relational database model (Star Schema) to analyze sales performance, profitability, and customer behavior effectively.

## 🏗️ Data Modeling (The Star Schema)
One of the core strengths of this project is the transition from a single large table to a structured relational model. I reorganized the data into:
* **Fact Table:** Containing all transaction metrics (Sales, Profit, Quantity, etc.).
* **Dimension Tables:** Dedicated tables for **Customers**, **Products**, **Geography (Regions)**, and **Segments**.
* **Benefits:** This modeling approach optimized the dashboard performance and allowed for deeper granular analysis across different business dimensions.

## 📊 Key Performance Indicators (KPIs)
The dashboard highlights 5 critical business metrics:
1.  **Total Customers:** Distinct count of unique customers (793).
2.  **Total Revenue:** Overall sales generated (~$2.3M).
3.  **Total Profit:** Net profit after all costs (~$287K).
4.  **Total Orders:** Number of successful transactions (9,993).
5.  **Items Sold:** Total quantity of products distributed (37,871).

---

## 📈 Dashboard Insights
* **Top Performers:** Analysis of the most profitable product categories and sub-categories.
* **Customer Segmentation:** Breakdown of sales by Consumer, Corporate, and Home Office segments.
* **Shipping Analysis:** Tracking the efficiency and volume of different ship modes (Standard Class being the most used).
* **Regional Performance:** Visualizing sales distribution across Central, East, South, and West regions.
